## AcademiSync 交互与 UI 草图（V1.0）

### 1. 整体布局

- **布局方式**：Streamlit 单页应用，采用「左侧侧边栏 + 右侧主内容区」结构。
- **页面标题**：顶部显示「📄 AcademiSync 文献综述生成」。

### 2. 侧边栏（左侧）

- **模块标题**：`配置`
- **控件列表**：
  - `智谱 API Key`：密码输入框
    - 默认值：从环境变量 `ZHIPUAI_API_KEY` 读取，如无则为空。
    - 用途：供 `generator.generate_review` 调用智谱 GLM-4。
  - `检索文献篇数`：滑块
    - 取值范围：3–20，默认 8。
    - 用途：控制 `researcher.fetch_papers` 调用中的 `limit` 参数。
  - `显示调试信息`：复选框
    - 选中时，在「调试信息」Tab 中展示原始返回 JSON 等细节。

### 3. 主内容区（右侧）

1. **话题输入区**
   - 组件：单行文本输入框
   - 标签：`输入研究话题：`
   - Placeholder 示例：`例如：Transformer 在医学影像中的应用`

2. **操作按钮**
   - 文本：`开始生成`
   - 类型：主按钮（primary）
   - 点击后执行完整流程：
     1. 校验 API Key 是否存在。
     2. 校验话题是否为空。
     3. 调用 `fetch_papers` 检索文献。
     4. 调用 `build_context_from_papers` 组装上下文。
     5. 调用 `generate_review` 生成综述。

3. **状态与错误反馈**
   - 使用 `st.spinner` 包裹完整调用链，提示文案如「📡 正在检索文献并生成综述，请稍候...」。
   - 常见提示：
     - Key 缺失：红色错误提示「❌ 缺少 API Key，请在侧边栏配置。」。
     - 话题为空：黄色警告「⚠️ 请输入话题。」。
     - 0 篇结果：黄色提示建议更换关键词或检查网络。

4. **结果展示 Tabs**
   - 使用 `st.tabs` 创建三个标签页：
     1. `📄 综述正文`
        - 内容：Markdown 渲染 `generate_review` 返回的综述文本。
        - 上方有小标题「✅ 生成结果」。
     2. `📚 文献列表`
        - 内容：遍历 `papers_meta`，逐条展示：
          - **[编号] 标题**
          - 摘要文本
          - 「原文链接」超链接（如 URL 存在）。
     3. `🔍 调试信息`
        - 内容：基础信息（话题、请求篇数、实际返回篇数）。
        - 当勾选「显示调试信息」时，展示原始 `papers` JSON，便于开发调试。

### 4. 辅助提示

- 页面底部展示网络相关提示，例如：
  - 「如果提示连接超时，请尝试关闭 VPN 或切换网络（智谱为国内 API 服务，某些网络环境可能无法访问）。」

该 UI 设计与 `app.py` 当前实现保持一致，后续如增加导出 Word、引用格式切换等功能，可在侧边栏增加开关和按钮，并在 Tabs 中扩展新的视图。
