## AcademiSync 产品定义（V1.0）

### 1. 产品定位

- **产品名称**：AcademiSync
- **一句话描述**：面向学生和科研新手的「半自动文献综述生成器」，帮助快速搭建综述初稿，同时保证引用真实存在的文献。
- **目标用户**：
  - 需要写课程论文、毕业论文的本科生 / 研究生
  - 需要快速扫盲新领域的工程师、产品经理
  - 希望减少「搜论文 + 读摘要 + 整理」机械劳动的科研人员

### 2. 核心价值

- **降低文献检索门槛**：用户只需输入研究话题，系统自动通过 Semantic Scholar 检索相关论文。
- **减少 AI 幻觉风险**：只允许基于真实抓取到的文献生成综述，Prompt 中明确「严禁编造文献」。
- **加速综述初稿产出**：在几分钟内生成包含背景、技术演进、对比分析和参考文献列表的 Markdown 草稿。

### 3. 使用场景（User Stories）

1. **课题开题前的快速扫盲**
   - 作为一名研究生，我要在一周内确定课题方向，希望快速了解「Transformer 在医学影像中的应用」有哪些代表性工作和常见研究路径。
2. **课程论文写作前的资料搜集**
   - 作为一名本科生，我要写一篇关于「大模型在教育中的应用」的课程论文，希望先有一版结构化的综述草稿，再针对感兴趣的部分深入阅读原文。
3. **跨领域工程实践前的信息收集**
   - 作为一名工程师，我准备在项目中引入某个前沿技术，希望快速了解其学术界的发展脉络和主流方案。

### 4. V1.0 功能范围

- **必须实现（MVP）**
  - 输入研究话题（支持中英文），可配置检索篇数。
  - 基于 Semantic Scholar 搜索真实文献，拉取标题、摘要、URL。
  - 将多篇文献整理为结构化上下文，并传给智谱 GLM-4 生成综述。
  - 在前端以 Markdown 渲染综述正文，并展示文献列表（带原文链接）。
  - 对常见错误（网络超时、Key 缺失、0 篇结果等）给出清晰提示。

- **不在 V1.0 中实现，但预留接口**
  - 导出 Word（调用 `python-docx`）。
  - 引用格式切换（APA / MLA / BibTeX）。
  - 本地 PDF 上传与解析，并与在线文献合并使用。

### 5. 非功能需求

- **易用性**：单页应用，主流程不超过 3 个交互步骤（输入话题 → 配置参数 → 生成）。
- **可解释性**：用户可以在「文献列表」Tab 中清楚看到所有被引用的文献。
- **安全性**：不在前端或代码库中暴露 API Key，统一通过 `.env` 或侧边栏输入。

### 6. 成功标准（V1.0）

- 至少 3 位目标用户可以在 10 分钟内：
  - 完成环境搭建与运行（pip 安装 + 配置 `.env`）。
  - 针对一个话题生成一篇结构完整的综述 Markdown。
  - 表述「对当前领域有一个大致的认识，可以继续深入某几篇文献」。
